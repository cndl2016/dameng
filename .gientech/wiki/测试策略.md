# 测试策略

## 测试现状

当前项目**未引入单元测试框架与自动化测试基础设施**。具体体现为：

- 所有模块（`ncdb-admin`、`ncdb-system`、`ncdb-common`、`ncdb-base`、`ncdb-job`、`ncdb-device`、`ncdb-ui`）的 `pom.xml` 均未声明 `spring-boot-starter-test`、`junit`、`mockito` 等测试依赖
- 所有模块均不存在 `src/test/` 目录及对应的测试源文件
- `ncdb-admin/pom.xml`(L104-L112) 明确配置了 `maven-surefire-plugin` 并设置 `<skipTests>true</skipTests>`，构建时跳过测试执行
- 前端 `ncdb-ui/package.json` 中未定义任何测试脚本或测试依赖（如 `vitest`、`jest`、`cypress`）
- 当前构建命令 `mvn clean package -DskipTests` 直接跳过测试阶段

> 来源：[`../../ncdb-admin/pom.xml`](../../ncdb-admin/pom.xml)(L104-L112)、[`../../AI-CONTEXT.md`](../../AI-CONTEXT.md)(L17)

## 质量保障方式

尽管缺少自动化测试，项目仍通过以下方式进行质量保障：

### 开发阶段验证

- **本地启动验证**：开发者在 IDE 中直接运行 `NcdbApplication.java` 启动类，通过 Swagger UI（`http://localhost:1081/swagger-ui/index.html`）或前端页面手动验证接口功能
- **前端开发验证**：前端通过 `npm run dev` 启动开发服务器，借助 Vite 热更新快速验证页面交互效果
- **数据库初始化验证**：通过执行 `doc/initDM.sql` → `initDMData.sql` → `initDM-DevEnv.sql` 的顺序脚本，验证数据库表结构与初始数据的正确性

### 部署验证

- **打包验证**：`mvn clean package` 构建生成 `ncdb-admin-package.tar.gz` 部署包，通过解压部署包并启动服务来验证
- **SSH 远程脚本验证**：`config/` 目录下的 Shell 脚本（如 `device_monitor.sh`、`alarm_message.sh`）通过 SSH 远程调用执行，在目标环境中验证设备管理功能

### 安全机制验证

- **Token 认证**：Spring Security + JWT 认证机制，Token 有效期 300 分钟，可通过接口测试工具验证
- **密码策略**：密码最大错误次数 5 次，锁定时间 10 分钟，登录时自动验证
- **验证码**：`math` 数学计算验证码，通过登录接口触发验证

> 来源：[`../../ncdb-admin/src/main/resources/application.yml`](../../ncdb-admin/src/main/resources/application.yml)、[`../../README.md`](../../README.md)(L4-L7)

## 测试建议

基于项目技术栈与模块结构，若计划引入自动化测试，建议按以下优先级推进：

### 优先级一：后端单元测试

| 建议项 | 说明 |
|--------|------|
| 测试框架 | JUnit 5 + Mockito |
| 测试范围 | 优先覆盖 `ncdb-common` 工具类与 `ncdb-system` 核心业务逻辑 |
| 核心测试点 | 用户管理、角色管理、字典数据、配置管理的 Service 层 CRUD 逻辑 |
| Maven 配置 | 在各模块 `pom.xml` 中添加 `spring-boot-starter-test` 依赖，并调整 `maven-surefire-plugin` 配置 |

### 优先级二：后端集成测试

| 建议项 | 说明 |
|--------|------|
| 测试框架 | Spring Boot Test + H2 内嵌数据库 |
| 测试范围 | 数据访问层（Mapper）、Controller 接口层 |
| 核心测试点 | 分页查询、动态数据源切换、达梦 DM8 兼容性验证 |
| 备注 | 项目已配置 H2 数据库依赖（`ncdb-common/pom.xml` L207-L210），可直接用于集成测试环境 |

### 优先级三：前端自动化测试

| 建议项 | 说明 |
|--------|------|
| 测试框架 | Vitest（单元测试）+ Cypress / Playwright（E2E 测试） |
| 测试范围 | 工具函数（`src/utils/`）、核心页面组件（登录、用户管理、设备管理） |
| 核心测试点 | 权限指令 `v-hasPermi`、字典数据回显、分页功能、文件上传 |

### 优先级四：接口与性能测试

| 建议项 | 说明 |
|--------|------|
| 接口测试 | Postman / Swagger 接口文档批量验证 |
| 性能测试 | JMeter 压测，重点关注 Tomcat 最大线程数（800）与连接排队数（1000） |
| 监控验证 | 结合 Prometheus + Grafana 监控组件验证系统稳定性 |

## 模块测试关注点

| 模块 | 测试关注点 | 涉及技术 |
|------|-----------|----------|
| ncdb-admin | 启动入口、Controller 聚合、Swagger 文档 | Spring Boot, Swagger 2 |
| ncdb-system | 用户/角色/菜单/部门 CRUD、字典管理、日志记录 | MyBatis, PageHelper |
| ncdb-common | 工具类、常量、异常处理、JWT 工具 | JWT, Hutool, ExpiringMap |
| ncdb-base | 告警消息、告警配置、软探针、WebSocket 推送 | WebSocket, SSH |
| ncdb-job | 定时任务调度、邮件发送 | Quartz, Spring Mail |
| ncdb-device | 设备管理、设备节点、SSH 远程调用 | SSH, Shell 脚本 |
| ncdb-ui | 页面渲染、路由权限、字典回显、文件上传 | Vue 3, Element Plus, Pinia |
