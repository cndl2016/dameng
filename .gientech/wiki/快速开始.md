## 简介

本快速入门指南旨在帮助新开发者快速搭建 NCDB 管理平台（manager）的本地开发环境。该系统是一个基于 **Spring Boot 2.6.15** + **JDK 17** 的单体架构后台管理系统，前端使用 **Vue 3 + Vite + Element Plus** 构建，数据库采用 **达梦数据库（DM8）**，支持 SSH 远程设备管理、告警监控、Prometheus 监控集成等核心功能。

> 来源：[README.md](../../README.md)、[pom.xml](../../pom.xml)(L15-L21)

## 系统要求

在开始之前，请确保您的系统满足以下要求：

- **JDK 版本**：JDK 17 或更高版本
- **构建工具**：Apache Maven 3.x
- **数据库**：达梦数据库 DM8
- **内存**：至少 8 GB RAM（服务节点资源限制）
- **磁盘空间**：至少 20 GB 可用空间（服务节点资源限制）
- **操作系统**：Windows / Linux（生产环境推荐 Linux）
- **网络**：稳定的互联网连接（用于下载依赖）

> 来源：[pom.xml](../../pom.xml)(L20-L21)、[application.yml](../../ncdb-admin/src/main/resources/application.yml)(L152-L156)

## 项目克隆

首先，将项目克隆到本地开发环境：

```bash
git clone https://github.com/cndl2016/dameng.git
cd dameng
```

## 环境准备

### 1. 安装 JDK 17

确保您的系统已安装 JDK 17 或更高版本：

```bash
java -version
javac -version
```

如果未安装，请从 [Oracle JDK 官网](https://www.oracle.com/java/technologies/javase/jdk17-archive-downloads.html) 或 OpenJDK 下载并安装。

### 2. 安装 Maven

项目使用 Maven 作为构建工具。请确保已安装 Maven 3.x 版本：

```bash
mvn -v
```

如果未安装，请从 [Maven 官网](https://maven.apache.org/download.cgi) 下载并配置环境变量。

### 3. 安装本地依赖（Framework JAR）

项目依赖自研的 `dm-framework` 组件，需要先手动安装到本地 Maven 仓库。执行以下命令：

```bash
mvn install:install-file -Dfile=framework/dm-framework/1.0.50/dm-framework-1.0.50.pom -DgroupId=com.dm.framework -DartifactId=dm-framework -Dversion=1.0.50 -Dpackaging=pom

mvn install:install-file -Dfile=framework/dm-framework-common/1.0.50/dm-framework-common-1.0.50.jar -DgroupId=com.dm.framework -DartifactId=dm-framework-common -Dversion=1.0.50 -Dpackaging=jar
```

> 来源：[README.md](../../README.md)(L12)、[pom.xml](../../pom.xml)(L121-L127)

### 4. 安装前端依赖

进入前端目录并安装依赖：

```bash
cd ncdb-ui
npm install
cd ..
```

> 来源：[ncdb-ui/package.json](../../ncdb-ui/package.json)

## 数据库初始化

### 1. 准备达梦数据库（DM8）

项目使用达梦数据库（DM8）作为主数据库。请确保本地已安装并启动 DM8 数据库服务。

默认数据源配置如下（详见 [application-druid.yml](../../ncdb-admin/src/main/resources/application-druid.yml)）：

| 配置项 | 默认值 |
|--------|--------|
| 驱动类 | `dm.jdbc.driver.DmDriver` |
| JDBC URL | `jdbc:dm://localhost:5236?schema=TEST_SCAFFOLD` |
| 用户名 | `SYSDBA` |
| 密码 | `Dameng@8888` |

> 来源：[application-druid.yml](../../ncdb-admin/src/main/resources/application-druid.yml)(L27-L38)

### 2. 执行初始化 SQL

在项目根目录的 `doc` 文件夹下，按以下顺序执行 SQL 脚本（开发环境）：

1. 首先执行 `doc/initDM.sql`——初始化基础表结构
2. 然后执行 `doc/initDMData.sql`——初始化基础数据
3. 最后执行 `doc/initDM-DevEnv.sql`——初始化开发环境数据

> 来源：[README.md](../../README.md)(L13)

## 构建和运行项目

### 构建项目

在项目根目录下执行以下命令构建整个项目：

```bash
mvn clean package -DskipTests
```

### 运行应用程序

有两种方式启动应用：

#### 方式一：在 IDE 中直接运行

在 IDE 中找到 `com.dm.cn.NcdbApplication` 主类，右键选择 **Run**。

#### 方式二：使用 Maven 命令运行

```bash
mvn spring-boot:run -pl ncdb-admin
```

### 启动前端开发服务器

在 `ncdb-ui` 目录下执行：

```bash
npm run dev
```

> 来源：[ncdb-ui/package.json](../../ncdb-ui/package.json)(L10)

### 验证应用启动

应用启动后，您应该能看到类似以下的日志输出：

```
Tomcat started on port(s): 1081 (http) with context path '/'
Started NcdbApplication in XX.XXX seconds
```

后端服务默认端口为 **1081**，上下文路径为 `/`。

> 来源：[application.yml](../../ncdb-admin/src/main/resources/application.yml)(L17-L22)

## 访问API文档

### Swagger UI

项目集成了 Swagger 2 用于 API 文档展示，需要在启动类上添加了 `@EnableSwagger2` 注解：

1. 启动应用后，在浏览器中访问：
   ```
   http://localhost:1081/swagger-ui.html
   ```

2. 也可以使用 Knife4j 或 Swagger Bootstrap UI（取决于项目配置），访问：
   ```
   http://localhost:1081/doc.html
   ```

### API 文档结构

- **基础路径**：`/`
- **端口**：1081
- **Swagger 路径映射**：`/dev-api`（见 [application.yml](../../ncdb-admin/src/main/resources/application.yml)(L121)）
- **文档路径**：`/swagger-ui.html`

## 故障排除

### 1. 端口冲突

如果端口 1081 被占用，可以在 [application.yml](../../ncdb-admin/src/main/resources/application.yml)(L17-L22) 中修改端口配置：

```yaml
server:
  port: 1082
```

### 2. 数据库连接失败

检查达梦数据库服务是否正常运行：

```bash
# 检查 DM8 服务状态（Linux）
systemctl status DmServiceDMSERVER
```

确认 [application-druid.yml](../../ncdb-admin/src/main/resources/application-druid.yml) 中的数据库连接地址、用户名和密码是否正确。

### 3. 本地依赖安装失败

如果 Framework JAR 安装失败，请检查 `framework/` 目录下的文件是否存在，并确认版本号与 `pom.xml` 中的 `dm-framework.version` 一致（当前为 `1.0.50`）。

### 4. 前端依赖安装失败

尝试清理缓存后重新安装：

```bash
cd ncdb-ui
rm -rf node_modules
npm cache clean --force
npm install
```

### 5. Maven 构建失败

清理并重新构建：

```bash
mvn clean install -DskipTests
```

如果私服依赖无法下载，请检查 [pom.xml](../../pom.xml)(L157-L163) 中的私服仓库地址配置是否正确。
