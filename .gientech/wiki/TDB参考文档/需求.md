# 需求文档：时序数据库运维网站 — TDB 可视化运维

## 1. 引言

### 1.1 功能概述

本需求旨在为 NCDB 管理系统增加 **达梦自研时序数据库（TDB）** 的可视化运维能力，包括 TDB 的版本管理、实例安装部署、启停、告警监控、备份还原、以及命令行终端等核心功能，通过 Web 界面实现对 TDB 单机/集群的全生命周期管理。

### 1.2 背景与目标

- **背景**：NCDB 管理系统已具备设备管理、系统监控、告警通知、SSH 远程执行等基础运维能力，但缺乏对时序数据库 TDB 的原生运维支持。时序数据库 TDB 完全兼容Postgresql的通信协议，可以用连接和操作pg的方式来使用它。
- **目标**：在现有 NCDB 单体架构上扩展 TDB 运维模块，使运维人员可通过 Web 界面完成对 TDB 的安装部署、配置管理、运行监控、故障处理等日常运维操作，降低运维复杂度，提升运维效率。

### 1.3 关键利益相关者与用户

- **运维人员**：需要直观、便捷地管理和监控 TDB 集群的日常运行状态。
- **系统管理员**：需要管理 TDB 的安装部署、配置变更、备份恢复等操作。
- **监控值班人员**：需要实时查看 TDB 集群告警信息，快速响应异常事件。

### 1.4 预期价值与收益

- 降低 TDB 运维门槛，通过可视化界面替代命令行操作
- 提升运维效率，标准化部署与运维流程
- 实现集中化监控告警，缩短故障发现与响应时间

## 2. 需求

### 需求 TDB-01：版本管理

1. 版本管理列表，支持增删改查、下载、启用禁用
2. 新增/修改版本，支持上传安装包文件

### 需求 TDB-02：实例管理

1. 实例管理列表，支持增删改查、查看任务日志（实例创建日志）、实例管理
2. 实例新增分单机和分布式两种： 
3. 单机版本：需要选择版本安装包，选择设备，填写安装目录，engine端口号，router端口号，rpc端口起始范围，并设置告警项（连接状态、CPU、内存、事务回滚率、总连接数、锁等待时间、缓存命中率等）
4. 分布式版本：可选多个设备，每个设备上部署一个节点，每个节点都平等的没有主从关系，其他都和单机版本一样
5. 实例创建完成后，可以点击列表页面的任务日志查看创建任务详细信息，包括进度百分比，状态（等待中、进行中、成功、失败）
6. 实例创建成功后，点击管理按钮跳转管理页面，管理页面可以查看节点列表，启动、停止节点，查看实例和节点的监控信息、告警信息，修改告警设置（告警通知方式），查看告警日志等

### 需求 TDB-03：告警管理

1. 通用的告警模板管理，配置告警项是否启用、触发告警阈值、告警通知方式（邮件、短信等）
2. 支持将告警模板应用到实例的功能
3. 支持查看告警日志

### 需求 TDB-04：实例监控

1. 监控单机/分布式实例的状态的和实例相关信息，有未启用、异常、健康等状态，按百分制打分

### 需求 TDB-05：数据操作

1. 可以连接运行中的实例，执行SQL查询（遵循pg规范），查看执行结果表格、执行失败报错原因。还能查看执行历史记录。

### 需求 TDB-06：导入导出

1. 使用 TDB 命令将数据库内的数据导出为文件，或者将数据文件导入 TDB
