# 编码指引

## 项目概述

本项目为 **达梦 NCDB 管理系统**（XXX 管理系统），采用单体架构，后端基于 Spring Boot 2.6.15 + Java 17，前端基于 Vue 3.4.31 + Vite 5.3.2 + Element Plus 2.9.3。项目通过 SSH 技术管理远程设备，依赖达梦 DM8 数据库。

### 3. 控制器层规范

#### 3.1 基础结构

- 所有 Controller 继承 `BaseController`
- 使用 `@RestController` + `@RequestMapping("{module}/{resource}")`
- 使用 `@Resource` 进行依赖注入（javax.annotation）

#### 3.2 权限控制

- 使用 `@RequiresPermissions("system:{resource}:{action}")` 注解标注接口权限
- 权限码格式：`{模块}:{资源}:{操作}`，如 `system:user:list`、`system:user:add`

#### 3.3 日志记录

- 使用 `@Log` 注解记录操作日志，指向 `@Log(title = "用户管理", businessType = BusinessType.INSERT)`

#### 3.4 API 文档

- 使用 `@ApiOperation(value = "接口描述")` 标注接口功能

#### 3.5 响应格式

- 列表查询返回 `TableDataInfo`（通过 `getDataTable()` 方法）
- 单条数据操作返回 `AjaxResult`（通过 `toAjax()/success()/error()` 方法）

#### 3.6 分页查询

- 在查询方法中调用 `startPage()` 开启分页，由 PageHelper 拦截实现

#### 3.7 参数校验

- 使用 `@Validated` + `@RequestBody` 进行请求参数校验
- 删除操作使用 `@PathVariable Long[] ids` 接收批量 ID

**完整示例**（来自 `SysUserController`）：

```java
@RestController
@RequestMapping("/system/user")
public class SysUserController extends BaseController {
    @Resource
    private ISysUserService userService;

    @RequiresPermissions("system:user:list")
    @GetMapping("/list")
    @ApiOperation(value = "获取用户列表")
    public TableDataInfo list(SysUser user) {
        startPage();
        List<SysUser> userList = userService.selectUserList(user);
        return getDataTable(userList);
    }

    @RequiresPermissions("system:user:add")
    @Log(title = "用户管理", businessType = BusinessType.INSERT)
    @PostMapping
    @ApiOperation(value = "新增用户")
    public AjaxResult add(@Validated @RequestBody SysUser user) {
        return toAjax(userService.insertUser(user));
    }
}
```

### 4. 服务层规范

#### 4.1 接口与实现分离

- 定义接口：`I{Resource}Service`
- 实现类：`{Resource}ServiceImpl implements I{Resource}Service`
- 实现类使用 `@Service` 注解

#### 4.2 依赖注入

- 使用 `@Resource` 注入服务层依赖
- 避免使用 `@Autowired`

#### 4.3 业务异常

- 使用 `ServiceException` 抛出业务异常，支持国际化消息

```java
throw new ServiceException(I18nMessageUtil.getMessage(
    MessageTipConstant.SYSTEM_USER_ADD_ACCOUNT_EXIST, user.getUserName()));
```

> 来源：[`../../ncdb-common/src/main/java/com/dm/cn/common/core/exception/ServiceException.java`](../../ncdb-common/src/main/java/com/dm/cn/common/core/exception/ServiceException.java)

### 5. 数据访问层规范

#### 5.1 MyBatis 配置

- Mapper 接口扫描：`@MapperScan("com.dm.cn.**.mapper")`
- Mapper XML 映射文件位置：`classpath*:mapper/**/*Mapper.xml`
- 类型别名包：`com.dm.cn.**.domain`

#### 5.2 MyBatis-Plus 分页

- 启用分页拦截器，自动识别 H2 或达梦数据库

```java
@Bean
public MybatisPlusInterceptor interceptor() {
    MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
    interceptor.addInnerInterceptor(new PaginationInnerInterceptor(
        dbTypeBean.isH2() ? DbType.H2 : DbType.DM));
    return interceptor;
}
```

### 6. 安全认证规范

#### 6.1 认证架构

- Spring Security + JWT Token 认证
- Token 自定义标识：`Authorization`
- Token 有效期：300 分钟
- 密码加密：BCryptPasswordEncoder

#### 6.2 权限校验

- 方法级注解：`@RequiresPermissions`、`@RequiresRoles`、`@RequiresLogin`
- 前端配合：`v-hasPermi="['system:user:add']"` 指令

#### 6.3 匿名访问

- 使用 `@Anonymous` 注解标记无需认证的接口
- 配置项：`/login`、`/register`、`/captchaImage` 等允许匿名访问

#### 6.4 安全配置

- 安全性配置位于 `SecurityConfig`，继承 `WebSecurityConfigurerAdapter`
- 基于 Token 的无状态会话（`SessionCreationPolicy.STATELESS`）
- 支持 CAS 单点登录（通过 `cas.enable` 配置开关）

> 来源：[`../../ncdb-system/src/main/java/com/dm/cn/system/config/security/SecurityConfig.java`](../../ncdb-system/src/main/java/com/dm/cn/system/config/security/SecurityConfig.java)

### 7. 日志规范

#### 7.1 操作日志（AOP）

- 自定义 `@Log` 注解，通过 `LogAspect` 切面实现
- 支持记录：标题、业务类型、操作人类别、请求参数、响应参数
- 敏感属性自动过滤：`password`、`oldPassword`、`newPassword`、`confirmPassword`
- 异步记录到数据库（`AsyncManager` + `AsyncFactory.recordOper()`）

#### 7.2 应用日志

- 使用 SLF4J + Logback
- 日志级别配置：
  - `com.dm.cn.base`: debug
  - `com.dm.cn.system`: debug
  - `org.springframework`: warn
- 后端日志路径：`/home/ncdb-admin/logs`

> 来源：[`../../ncdb-admin/src/main/resources/application.yml`](../../ncdb-admin/src/main/resources/application.yml)(L34-L39)、[`../../ncdb-system/src/main/java/com/dm/cn/system/config/aspectj/LogAspect.java`](../../ncdb-system/src/main/java/com/dm/cn/system/config/aspectj/LogAspect.java)

### 8. 常量与枚举规范

#### 8.1 常量类

- 统一放置在 `com.dm.cn.common.core.constant` 包
- 使用 `public static final` 声明，提供 Javadoc 注释
- 按用途分类：`Constants.java`（通用）、`UserConstants.java`（用户）、`TokenConstants.java`（Token）

#### 8.2 枚举定义

- 统一放置在 `com.dm.cn.common.core.enums` 或 `com.dm.cn.common.enums` 包
- 枚举命名：`{Status}Enum`（如 `UserStatus`、`BusinessType`、`OperatorType`）

```java
public enum OperatorType {
    OTHER,      // 其它
    MANAGE,     // 后台用户
    MOBILE      // 手机端用户

```

### 9. 配置管理规范

- 配置文件：`application.yml`（主配置）、`application-druid.yml`（数据源）
- 敏感信息加密：Jasypt（`PBEWithMD5AndDES` 算法），使用 `ENC(...)` 包裹
- 配置属性前缀：`manager`（项目）、`token`（认证）、`ncdb`（路径）、`ssh`（SSH）、`cas`（CAS）
- 配置文件外部化：部署时 `config/` 目录下的配置优先于包内配置

> 来源：[`../../ncdb-admin/src/main/resources/application.yml`](../../ncdb-admin/src/main/resources/application.yml)

### 10. 国际化规范

- 资源文件路径：`i18n/messages`（Spring MessageSource）
- 工具类：`I18nMessageUtil` + `MessageUtils`
- 支持中英文切换（`zh-CN` / `en-US`），通过用户语言字段存储

### 3. API 调用规范

#### 3.1 请求封装

- 统一通过 `src/utils/request.js` 封装的 Axios 实例发起请求
- 每个模块在 `src/api/` 下创建独立的 API 文件

#### 3.2 命名规范

- 导出函数命名：`list{Resource}`（查询列表）、`get{Resource}`（查询详情）、`add{Resource}`（新增）、`update{Resource}`（修改）、`del{Resource}`（删除）

```javascript
// src/api/system/user.js
import request from '@/utils/request'

export function listUser(query) {
  return request({ url: '/system/user/list', method: 'get', params: query })
}
export function addUser(data) {
  return request({ url: '/system/user', method: 'post', data: data })
}
export function updateUser(data) {
  return request({ url: '/system/user', method: 'put', data: data })
}
export function delUser(userId) {
  return request({ url: '/system/user/' + userId, method: 'delete' })
}
```

> 来源：[`../../ncdb-ui/src/api/system/user.js`](../../ncdb-ui/src/api/system/user.js)

## 开发工作流

### 1. 环境要求

### 2. 本地开发流程

```bash
# 1. 安装本地框架依赖
mvn install:install-file -Dfile=framework/dm-framework/1.0.50/dm-framework-1.0.50.pom \
  -DgroupId=com.dm.framework -DartifactId=dm-framework -Dversion=1.0.50 -Dpackaging=pom
```

## 代码质量与最佳实践

### 1. 通用规范

- 使用 `@author` 标注代码作者和日期
- 关键类和公共方法添加 Javadoc 注释
- 避免硬编码，使用常量/枚举替代魔法值
- 使用 `StringUtils`、`DateUtils` 等工具类处理字符串和日期

### 2. 异常处理

- 业务异常使用 `ServiceException`，支持国际化消息
- 全局异常统一由 `GlobalExceptionHandler` 处理
- 敏感信息（密码、Token）不允许出现在日志或响应中

### 3. 安全规范

- 密码使用 `BCryptPasswordEncoder` 加密存储
- 配置文件中敏感信息使用 Jasypt 加密
- 接口权限通过 `@RequiresPermissions` 严格控制
- 前端页面按钮通过 `v-hasPermi` 指令控制可见性

### 4. 性能规范

- 数据库查询使用 PageHelper 分页，避免全量加载
- 使用 `AsyncManager` 异步处理非核心任务（如操作日志记录）
- SSH 连接使用连接池管理，避免频繁创建销毁
